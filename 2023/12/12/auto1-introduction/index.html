1. brief introduction to AutoFPGA

Problem Author himself came acrossed

Author getting annoyed that it felt like he kept ==rebuilding the wheel==. Each project needed to have a bus designed for it, it needed to have interrupts defined and assigned, peripherals needed to be integrated, and all these things changed from project to project.

Problem other students came acrossed

Many students declare that they want to build a design in Vivado , but what they really mean is that they want to build a design using the ==graphical design method== Vivado supports.. From author’s standpoint, it seems like the biggest problem these new FPGA students are struggling with is the ==graphical design flow that they are so eager to learn==.

为什么不使用图像化设计呢，作者进行如下的论述：

  • ==1.The design flow hides a lot of configuration detail.== 使用图形化的参数配置隐藏了大多数的细节，方便了设计，但是也带来了问题；

  • ==2.The design flows aren’t textual. You can’t do a “diff” between two designs.== This makes it hard to answer the question of “how is my design different from the example/reference design.”

    I guess I hadn’t realized how much of a drag many of the modern graphical tools were becoming to the professional designer. This may be because GUI tend to start up slowly. It may be because GUI tend to limit your design options to the road most people travel upon . However, for me the problem is twofold:

    • First, a GUI forces me to slow down and use a mouse.
    • Second, building a design can take a lot of work . The option of using make will only rebuild the code that had been changed，it saves time.

  • ==3.It isn’t clear how to place a scope or a probe into such a design, to see and understand what is going on within it.== 图形化设计中的代码一旦出现错误，无法使用探针去了解哪里出了问题，据我所知，大部分人他们的处理方式只有——使用 ILA 工具抓取波形，然后再去肉眼观察哪里出错了。*Going back to the individual struggling to get HDMI video working, he didn’t have any clear locations within his design where he could ==create a tap and see the problem.==

  • ==4.Once use graphical design methods, there doesn’t seem to be any way to “take the training wheels off” of these methods==, so as to be able to dig down into the nuts and bolts of a design once you were ready to do so.

    长时间的使用这些辅助工具，当你真正的想要深入一个结构的内部时，却无从下手！用的越多，欠缺的越多。虽然这个工具方便，但是在剥夺我们的创造性。

  • ==5.the Xilinx toolchain in particular has created a horrendous problem for Digilent ’s support staff, in that every new version of Vivado breaks their demonstration projects.==

    每一个新版本的Vivado都会破坏他们的演示项目。这里的“破坏”可能意味着新版本的Vivado与旧的演示项目不兼容，导致项目无法正常运行或者功能出现问题。经典的面相 EDA 厂商编程。>

Any design flow or tool suite that hides “useless” or “irrelevant” details will also hide the location of your bug

2. AutoFPGA’s Goals

goals

The goal for AutoFPGA was simple: to be able to run AutoFPGA with a list of peripherals, and have it automatically connect those peripherals together in a manner prescribed by their configuration files.

look in an example Makefile that calls AutoFPGA, you’ll find a line that just calls AutoFPGA with a list of components.

features：

  • “Make“able：

    type “make” in a top level directory, have make recurse(递归，这里指make去其他目录继续make) as necessary into any sub-directories to process component configuration files, build design files, assemble those files together, and build a project all from one make command.

  • No new language, just key=value pairs：

    Further, author don’t really want to create a new language for component interaction and connection.For this reason, AutoFPGA is primarily a Copy and Paste utility. The configuration files will identify what things need to be copied and pasted where to create the desired output files.

    The files consist of a simple series of ==“@KEY=VALUE” pairs==, where the VALUEs can take multiple lines–often containing pieces of Verilog or C++ code. 可以简单的理解为，替换的键值对。

  • No hidden design components

    AutoFPGA just creates text files. Since AutoFPGA is primarily a copy and paste utility, comments within the AutoFPGA configuration files will just get copied into their respective places.

    What this means is, when you have a problem with the design of one of your components, you can look back into that components configuration file and see what got copied and adjust it.

3. A Simple AutoFPGA Design

Author have created a new project, an AutoFPGA Demo, which contains a series of simple peripherals that can easily become a part of any project using AutoFPGA. This should be simple enough for any new student to learn how to build and integrate components together using AutoFPGA.

See it at : An AutoFPGA peripheral demonstration project.It is designed to show users how simple peripherals can be incorporated into a design via AutoFPGA. The plan is to describe how various components can be added (or removed) from a design using this design as an example.
