2. NO burst

Option for no bursts, only single transactions

有个问题，我没有弄清楚，f_axi_rd_nbursts 、f_axi_rdid_nbursts 、f_axi_rdid_nbursts 三者之间的关系。

猜测变量可能有以下含义：

  • f_axi_rd_nbursts：可能表示当前的突发数量。
  • f_axi_rdid_nbursts：可能表示当前的读取突发数量。
  • f_axi_rdid_ckign_nbursts：可能表示正在检查的突发数量。

这些变量之间的关系可能取决于你的设计和代码的具体逻辑。例如，当一个新的突发开始时，f_axi_rd_nbursts可能会增加。当一个突发被读取时，f_axi_rdid_nbursts可能会增加。当一个突发被检查时，f_axi_rdid_ckign_nbursts可能会增加。

3.2 check length of read burst

关于什么时候会 check this read burst需要后面更多了解。

f_axi_rdid_nbursts信号寄存的是 read operation 下 rd_id 相应的应该接受的 number of burst。当且仅当 R channel 进行正确的 ID 读取 the last beat of one burst ，将相应的 read_id_number_of_burst 数量减 1。 将结果存储在 f_axi_rdid_ckign_nbursts 寄存器中。

f_axi_rdid_ckign_outstanding 寄存器的作用和 rdid_nburst的作用相当。由减小bumber_of_burst 更改为见小 number_of_outstanding。

这里需要注意 write_outstanding 和 number_of_burst 意思差不多。但是在这里 read_oustanding 和 write_pending 意思差不多，但又不完全一样， read_outstanding 表示所有 read_burst 剩余的所有 number_of_data that need to be read。

when we decide to check the length of one burst, we should store the AR_len signal data in formal_regsiter name f_axi_re_cklen right now.

3.3 check length of return burst

1. logic & behavior

处理一个返回，更新突发长度，并在突发结束时停止检查.

这里的 f_axi_rd_cklen 区别于 f_axi_rd_outstanding ，前者是一次 burst 剩余的需要传输的 data ，后者是所有 burst 剩下的需要传输的 number_of_data。

综上所述，这个逻辑只是为了在 master 获取 r_cahnnel 最后一个数据的时候处理3个事情：

  • when accept the last data from slave , the wlast signal must be asserted
  • everytime accept one beat from slave , the reset lenfgth of beats should minus one
  • one accept the last beat , deassert the formal_check_vlaid signal
