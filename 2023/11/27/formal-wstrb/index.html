1. Desc & Purp

Purpose:

This file is an adjunct to the formal properties for AXI. It contains a check of the write strobe signals, which can be used in a lighter check than the entire properties of the AXI bus. It’s useful for checking write packets in flight.

“write packets in flight” 意味着在数据包从源端到目的端传输的过程中进行检查，而不是在数据包到达目的端后进行检查。

2. Param & Signal

参数：

  • parameter C_AXI_DATA_WIDTH = 128：定义了AXI数据的宽度，这里设置为128位。
  • localparam DW = C_AXI_DATA_WIDTH：局部参数，其值等于C_AXI_DATA_WIDTH。
  • localparam LGBYTES = $clog2(DW/8)：局部参数，表示每个数据字包含的字节数。

输入/输出信号:

  • input wire [6:0] i_addr：宽度为*[6:0]*，表示地址。
  • input wire [2:0] i_size：宽度为*[2:0]*，表示突发大小。
  • input wire [DW/8-1:0] i_wstrb：宽度为[DW/8-1:0]，表示哪些字节应该被写入。
  • output reg o_valid：定义了一个输出信号o_valid，表示请求是否有效。

如果数据宽度DW不足以支持突发大小i_size，那么这段代码会将strb_mask的所有位设置为 1。这是因为在这种情况下，所有的数据位都应该被写入，所以所有的选通信号都应该是有效的。

这里如果 DW 位宽过小，可能会对一些不存在的 bit 进行逻辑处理，所以需要调用 verilator 的 lint_off SELRANGE 方法消除相关的wanring ，在结束之后重新 lint_on SELRANGE。

3.2 Addr & Mask

在 3.1 小结中展示了数据位宽 DW 和 write channel strb之间的关系，很好理解，需要对传输数据的每一个 byte 生成一个对应的 strb 数据。那么为什么 address 会和 mask 产生关系呢？

在axi 协议中要求数据和地址对齐，具体可以查看相关的文章内容：Understanding AXI Addressing (zipcpu.com)

对于网络上的信息我感觉不太能理解，现在做一个实验如下：

假设我有一个axi 数据位宽是 DW 等于 32bits (4 bytes)，根据3.1 小结的逻辑， strb_mask 应该表示为 1111。现在对 i_addr[1:0]做分类讨论：

  • 如果 i_addr[1:0] ==2‘b00，表示目前地址和数据位宽已经align，可以将所有的数据写入；
  • 如果i_addr[1:0] ==2‘b01，表示目前地址和数据位宽没有align，需要左移 1 byte，只有 3 byte 能够对齐写入；最低位因为不对齐，并不能写有效。
  • 如果i_addr[1:0] ==2‘b10，表示目前地址和数据位宽没有align，需要左移 2 byte，只有 2 byte 能够对齐写入；
  • 如果i_addr[1:0] ==2‘b11，表示目前地址和数据位宽没有align，需要左移 3 byte，只有 1 byte 能够对齐写入；

3.3 word aligned strobe

Purpose : Create a word aligned strobe by shifting the word_mask by an aligned address.创建一个与地址对齐的写入选通信号（Write Strobes）

1. Why word_mask

strb_mask 是两个信号的与结果—— word_mask & strb_mask；word mask 通常用于指示数据总线上的哪些字（通常为 32 位）是有效的。这样，通过结合使用 word mask 和 strb mask，我们可以非常精确地控制哪些数据应该被写入内存，从而实现对数据传输的精细控制。

举例说明，如果我有一个128 bits 位宽的 axi DW 通道，但是 size 的结果大小等于 010 ，这是一个 narrow transfer ，16 bytes 的数据传输位宽，但是每次数据传输只有 one word (4 bytes) 的最大有效数据。

  • 如果 strb_mask == 16‘1110_0000_0000_000 ; 表示最高的 one word中有 3 byte 的有效数据；
  • 那么计算出来的word_mask必须是 16‘1111_0000_0000_000 ，表示最左侧的 word 中会有有效数据；
  • 两个数据的&结果，则是最终的strb数据。这么做无非是给计算加上一层保险。
