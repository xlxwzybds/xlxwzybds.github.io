1. Desc & Purp

在 axi 协议中有很多复杂的特性，在分类 formal中我将把这些特性tear up ，尽量一个个分开描述，方便掌握。

Description: The AXI (full) standard has some rather complicated addressing modes, where the address can either be FIXED, INCRementing, or even where it can WRAP around some boundary. When in either INCR or WRAP modes, the next address must always be aligned. In WRAP mode, the next address calculation needs to wrap around a given value, and that value is dependent upon the burst size (i.e. bytes per beat) and length (total numbers of beats).

Purpose: Since this calculation can be non-trivial, and since it needs to be done multiple times, the logic below captures it for every time it might be needed.

2. signals

The parameter we only need is just the address_width of axi_address channel. About the input signals , we have four in the whole. They are i_last_addr, i_size , i_burst , i_len.

  • [2:0] i_size 参数意味着axi bus data size，分别可以设置为：1byte、2byte、4byte……128byte；
  • [1:0] i_burst：用于表示突发模式（例如，fixed, incr, wrap, reserved）。
  • [7:0] i_len：表示突发传输的beat 长度 i_len+1，可以表示的范围是0–255.

输出信号o_incr表示地址增量， o_next_addr表示计算出的下一个地址。

宽度为AW的寄存器。(* keep *)是一个指示，告诉编译器在优化过程中保留这些寄存器。

3. Implement

分为 increment、 wrap_mask 、 o_next_addr、 o_incr四个部分。

3.1 increment

increment是之前 declare 的一个寄存器，用于存放每次 beat transfer需要增加的 address byte 数量。

i_burst是 burst 类型输入信号，如果 burst == 0意味着这个传输是 fixed type ，如果非零，则会进行地址上的变化。

sth about the usage of tool named verilator

在赋值的过程中， i_size 的不同大小，对 increment 赋值的数据宽度大小都不一样，会触发 verilator的宽度警告。所以需要使用 // verilator lint_off WIDTH关闭宽度警告，使用 // verilator lint_off WIDTH重新打开宽度警告。

3.2 wrap_mask

如果 i_burst==2‘b10将会启用这一个模块，代表使用了 wrap type的计算address方式。

wrap_mask在处理wrap类型的突发传输时起到关键作用。在wrap类型的突发传输中，地址会根据i_size的值进行增加，但是当达到某个上限地址时，地址会回绕到一个较低的地址。假设i_size为2（表示每次传输4字节），i_len为3（表示突发长度为4），那么wrap_mask的值将被计算为

$$
2^{2+2}-1 =15 = 0XFFFF
$$

3.3 next_addr

verilator lint_off / lint_on SELRANGE

选择范围警告通常在你试图访问一个超出其定义范围的位时触发。例如，如果你有一个8位的信号signal，然后试图访问signal[9]，那么你就会得到一个选择范围警告，因为你试图访问一个不存在的位。这里为什么会用到这个命令？

首先是根据 data_size 对地址进行 byte align！

AW是一个参数，表示地址的宽度。如果AW的值小于i_size，那么在访问o_next_addr的某些位时就可能超出范围，从而触发选择范围警告。这个还是十分重要的，一旦 AW 参数设置过小，就会触发错误

如果是wrap type 类型。则需要获取相应的高位，与next_addr 的地位进行拼接。当地址增加到上限时，可以将地址回绕到一个较低的地址，而不是继续增加到一个不存在的地址。这对于处理wrap类型的突发传输是必要的，因为在这种突发传输中，地址需要在达到上限后回绕到较低的地址。

3.4 incr

[7:0] o_incr每次对外输出one beat 或者 one transfer 需要增长的地址大小。i_size表示的是每次传输的数据大小，其最大值为7，对应的数据大小为27，即128字节。这与代码中7:0的选择范围是一致的，因为7:0表示的是8个位，对应于一个字节。所以，当i_size为7时，一次传输的数据大小为128字节，这就需要8个位来表示。

4. conclusion

​ 本文粗略地对 axi 协议中的 address increment 相关问题进行了讨论。在第一、二章节中表述博客写作的目的，对相关信号进行回顾。在第三节中分为三个主要部分，对每次地址变化的 increment、 wrap mask 、 next_addr三个寄存器相关工作逻辑进行了梳理。
